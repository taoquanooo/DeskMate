# DeskMate 宠物缩放与动态预览设计

## 目标

修复桌面宠物缩放后被窗口裁切的问题，将尺寸范围扩展为 25%–300%，支持滑杆和百分比输入，并让设置首页左侧预览以固定尺寸展示当前选中的动态宠物。

## 范围

本次改动只涉及宠物尺寸、原生窗口几何、首页预览和相关测试。不会改变宠物图集格式、移动速度、行为优先级、提醒功能或自定义宠物目录结构。

上一轮尚未提交的自定义宠物图集运行时授权修复需要保留，并在后续实现提交中一并验证。

## 尺寸设置

- 桌宠缩放范围为 25%–300%，默认 100%。
- 滑杆范围为 25–300，步进为 5%。
- 滑杆旁增加百分比数字输入框，允许输入整数百分比。
- 数字输入在按回车或失去焦点时提交，避免用户尚未输完时连续改变桌宠窗口。
- 空值或非数字恢复为当前已保存值；小于 25 的值保存为 25，大于 300 的值保存为 300。
- 前端设置合并与 Rust 本地设置清洗都使用 0.25–3.0 的相同边界，旧设置无需迁移。

## 桌宠窗口缩放

原生宠物窗口继续使用 `192×208 × scale` 的逻辑尺寸。前端 `.pet-window` 不再固定为 192×208，而是填满原生窗口，并用底部居中的布局承载固定单元格尺寸的 `PetSprite`。`PetSprite` 仍以底部中心为缩放原点，因此视觉边界与原生窗口边界一致，放大和缩小时都不会在窗口内部被裁切。

改变尺寸时，以旧窗口的底部中心为锚点计算新位置，使宠物看起来在原地变大或变小。新位置需要限制在当前显示器的工作区内，避免超出屏幕或压到任务栏。若无法识别当前显示器，则只调整窗口尺寸，不中断设置保存。

窗口几何计算提取为不依赖 Tauri 的纯函数，输入旧位置、旧尺寸、新尺寸和工作区，输出新位置，便于覆盖负坐标、多显示器和边界夹取测试。

## 动态预览

设置首页预览显示当前实际选中的宠物，包括内置、官方下载和本地自定义宠物。它通过现有 `pet_current` 命令获得初始图集路径和图集版本，并监听 `pet://changed` 立即切换。

预览尺寸固定为 100%，不读取 `settings.pet.scale`，所以调整桌面宠物大小不会改变预览。预览按固定演示序列循环：休息、挥手、跳跃，再回到休息。Codex v1 图集仍使用现有兼容映射；演示不使用注视状态。

预览只负责展示：不注册点击、双击、拖动、鼠标注视或运行时交互事件。组件卸载时停止计时器和事件监听。

身份区域从现有内置信息、本地扫描结果或官方目录中查找当前宠物名称；查不到时显示 ID 与版本，不伪装成内置“默认伙伴”。图集读取失败时保留上一个可用预览，初次失败则使用内置宠物作为兜底。

## 组件与数据流

- `PetSizeSetting` 负责滑杆、百分比草稿值、提交和边界处理。
- `PetPreview` 负责当前宠物载荷、非交互演示时间线和 `PetSprite` 渲染。
- `PetSettings` 只组合预览与设置项，不自行维护动画计时器。
- `settings_patch` 清洗并保存缩放值，然后调用 `apply_window_settings`。
- `apply_window_settings` 计算新尺寸与位置、调整原生窗口，最后广播 `settings://scale` 供真实桌宠更新视觉缩放。

所有尺寸更新仍通过现有设置保存链路，不增加新的 Rust command 或持久化字段。

## 错误处理

- 无效百分比输入不写入设置，并恢复为当前值。
- 原生窗口尺寸或位置调整失败不会破坏已经保存的其他设置；失败只影响当次视觉调整。
- 预览宠物载荷或本地图集加载失败时保留上一可用预览，不让设置页崩溃。
- 25%–300% 边界在前端和 Rust 两层执行，防止损坏或手工修改的 JSON 造成异常窗口尺寸。

## 测试与验收

- 设置合并和 Rust 设置清洗分别验证 25% 与 300% 边界。
- 百分比输入验证回车、失焦、空值、越界值和滑杆同步。
- 预览验证不使用桌面宠物 `scale`、会随时间切换帧、会响应 `pet://changed`，且不产生交互事件。
- 窗口几何纯函数验证放大、缩小、工作区夹取和负坐标显示器。
- 现有前端测试、类型检查、格式检查、宠物校验和生产构建必须通过。
- GitHub Actions Windows Rust 测试与安装包构建必须通过。
- 人工验收需在 25%、100%、300% 下确认宠物完整可见，并测试靠近屏幕四边、任务栏及另一块显示器时的缩放。
