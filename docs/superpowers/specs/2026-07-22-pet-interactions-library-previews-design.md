# DeskMate 宠物交互与宠物库预览设计

## 目标

增强关闭自动漫游时的桌宠反馈，并让宠物库中的内置、自定义和官方宠物都有静态预览。交互需要在 Windows 原生窗口拖动、多显示器和不同 DPI 下保持稳定，宠物库预览不得持续播放动画或显著增加常驻资源占用。

## 交互映射

- 单击桌宠：播放跳跃，持续约 1 秒。
- 双击桌宠：播放挥手，持续约 0.9 秒。单击会延迟约 230 毫秒确认，以免双击先触发跳跃。
- 右键桌宠：从挥手、等待、完成检查中随机选择一个动作，并阻止浏览器右键菜单。
- 关闭自动漫游时拖动桌宠：根据原生窗口的横向位置变化播放向左或向右奔跑；竖直拖动沿用最近一次水平方向，尚无方向时保持 idle。
- 松开鼠标：停止奔跑并恢复后端原本希望播放的状态，通常为 idle。
- 开启自动漫游时，后端已有的漫游动画仍负责运行方向；本次拖动方向反馈只在关闭自动漫游时启用。

动作优先级固定为：原生拖动 > 单击、双击、右键交互 > 后端提醒或更新状态 > 漫游 > 注视 > idle。拖动期间清除待执行的单击动作，避免松手后误跳。

## 原生拖动方向

采用后端窗口位置检测方案。`MotionEngine` 已以 30Hz 运行并持有 `dragging` 状态，因此在拖动期间读取宠物窗口的物理坐标，与上一个采样位置比较：

- 横向变化超过小阈值时发出对应的 `running-left` 或 `running-right`。
- 方向未变化时不重复发事件，避免前端动画反复从第一帧开始。
- Windows 左键释放后，后端清除拖动状态并发出恢复动画事件。
- 每次拖动结束时清空位置和方向缓存，下一次拖动重新判断。

方向判定与 Tauri 窗口事件解耦为纯函数，覆盖左移、右移、小幅抖动和无历史位置的单元测试。

## 静态宠物预览

新增可复用的 `PetThumbnail` 组件，所有预览使用统一尺寸容器：

- 内置默认伙伴：使用内置 `spritesheet.webp` 的 idle 第一帧。
- 自定义宠物：扫描结果新增经过校验的 `spritesheetPath`；Rust 仅把已验证的具体 WebP 文件加入 Tauri asset scope，前端显示对应图集的 idle 第一帧。
- 官方目录宠物：优先使用目录中的 HTTPS `previewUrl`。图片加载失败时显示统一宠物占位图。

自定义图集通过现有 `PetSprite` 的 v1/v2 背景尺寸逻辑显示，但不启动计时器，因此只是静态第一帧。缩略图容器负责裁切和缩放，不修改桌面宠物的实际大小设置。所有图片提供宠物名称的无障碍标签。

## 数据接口

`LocalPetV1` 增加：

```ts
spritesheetPath: string;
```

Rust 本地宠物扫描载荷增加同名字段。该路径只在文件已存在、清单有效、WebP 尺寸和 Codex 图集结构通过校验后返回并授权。官方目录继续使用现有 `previewUrl`，不改变目录格式。

## 错误处理与性能

- 本地缩略图加载失败时显示占位图，不影响“使用”按钮和其他宠物。
- 官方预览下载失败时显示占位图，不阻塞宠物库页面。
- 缩略图无动画定时器；列表中的资源由 WebView 正常缓存。
- 扫描仍逐个报告无效宠物，未通过验证的文件不会获得 asset scope 权限。

## 测试与验收

- 前端测试单击、双击、右键动作，以及拖动开始后取消待执行点击。
- Rust 纯函数测试窗口位置变化到左、右和抖动阈值的方向判断。
- 前端测试内置、本地和官方三类缩略图来源，以及图片失败占位回退。
- Rust 测试本地扫描只为有效宠物返回图集路径。
- 运行完整 `pnpm verify`、Rust 测试和 GitHub Actions Windows x64 安装包构建。

验收时，关闭自动漫游并横向拖动桌宠可立即看到对应奔跑动作；松手后停止。宠物库每个有效条目都有静态预览，图片失败不会破坏页面或宠物切换。
